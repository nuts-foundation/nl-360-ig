Volume 3 of this specifcation describes the agreements and specifications about content.

# Data set definitions

The 360° specifications use the healthcare information models (HCIM's, also called "zibs") and FHIR specifications
described as specified in the following table.

Resources are retrieved based on the **FHIR profile** (`meta.profile`) through the `_profile` search parameter, no
longer based on SNOMED/LOINC codes. To support different zib versions, each query accepts multiple profile canonicals as
an OR list (comma-separated): the R4 `nl-core` profile and the STU3 `zib2017` profile. The canonicals are given without
`|version`, so that all published versions of a profile match.

> **Note — preconditions for `_profile` searching**
> - Source systems must populate `meta.profile` on their resources with the canonical(s) listed here; if they do not,
    the query returns nothing.
> - A few zibs do not (yet) have an `nl-core` R4 profile in the version used (`nictiz.fhir.nl.r4.nl-core`
    0.12.0-beta.4); for those only the STU3 profile is listed. This is flagged per row.
> - The medication zibs (Medicatieafspraak, MedicatieGebruik, Toedieningsafspraak) have their R4 profiles in a separate
    medication package that is not yet included as a dependency.
